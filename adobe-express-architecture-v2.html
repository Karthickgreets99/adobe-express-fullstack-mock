Interview talking points — say these out loud
Storage "All S3 uploads are private ACL by design. Users never get a direct S3 URL — they get a signed URL via CloudFront with a 1-hour expiry. If the URL leaks, it expires."
Caching "Redis sits between the API and MySQL. At 116,000 read requests per second, without caching MySQL would fall over. 80% cache hit means most reads never touch the database."
Scale "Write QPS is ~1,160/sec which a single MySQL cluster handles fine. Read QPS at 116K/sec is why we need Redis and read replicas — that's the 100:1 read/write ratio."
IaC "Every resource is provisioned via CloudFormation — environment-aware, versioned, repeatable. Dev gets a single-AZ RDS, prod gets Multi-AZ with deletion protection. Same template, different parameters."
Security "Two SQL injection fixes in the codebase — all queries are parameterised. bcrypt with 12 salt rounds for passwords. JWT tokens validated at the API Gateway before any service is reached."
